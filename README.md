# منظومة متابعة مهام البنية التحتية

تطبيق ويب ثابت لإدارة ومتابعة مهام البنية التحتية والاجتماعات والجهات والمستخدمين. يعمل كتطبيق Front-End فقط باستخدام HTML وCSS وJavaScript وIndexedDB داخل المتصفح.

## طريقة التشغيل

افتح ملف `index.html` مباشرة في المتصفح. يمكن كذلك رفع المجلد كما هو إلى GitHub Pages أو أي استضافة Static Hosting.

## بيانات الدخول الافتراضية

- اسم المستخدم: `admin`
- كلمة المرور: `ChangeMe@123`
- الدور: مدير النظام

يوجد أيضًا سبعة حسابات فرعية افتراضية: `user1` إلى `user7`، وكلمة مرور كل حساب `ChangeMe@123`.

## الوظائف الرئيسية

- حفظ تلقائي داخل IndexedDB باسم `InfrastructureTasksDB`.
- إدارة المهام مع البحث والفلاتر والحذف الناعم وسجل تدقيق.
- إدارة الاجتماعات الناتجة من المهام.
- إدارة المستخدمين والصلاحيات والجهات.
- لوحة مؤشرات ورسوم بيانية باستخدام Chart.js عند توفرها.
- تصدير المهام إلى Excel باستخدام SheetJS عند توفرها.
- استيراد Excel مع شاشة مراجعة قبل التنفيذ.
- تحميل نسخة احتياطية JSON واستعادتها.

## استيراد Excel

من صفحة "قاعدة البيانات"، اختر ملف Excel. يحاول التطبيق اكتشاف صف العناوين تلقائيًا ويدعم الأعمدة التالية عند وجودها:

- الإفادة
- الحالة
- حالة التصعيد
- التاريخ
- المسؤول
- المهمة المسندة
- م
- محضر اجتماع
- عدد المهام
- تاريخ الاجتماع
- جهة الاجتماع

بعد قراءة الملف تظهر مراجعة بعدد الصفوف والمهام الجديدة والمحدثة والبيانات الناقصة. اختر وضع الاستيراد ثم نفذ العملية.

## تصدير Excel

من صفحة المهام يمكن تصدير النتائج الحالية بعد الفلترة. إذا لم تتوفر SheetJS بسبب عدم وجود اتصال بالإنترنت، يتم تصدير CSV كبديل.

## النسخ الاحتياطي والاستعادة

صفحة "قاعدة البيانات" توفر:

- تحميل نسخة احتياطية كاملة بصيغة JSON باسم `infrastructure_tasks_backup_YYYY-MM-DD.json`.
- رفع ملف JSON ومراجعة ملخصه قبل الاستعادة.
- الاستعادة تستبدل بيانات IndexedDB الحالية بعد التأكيد.

## النشر على GitHub Pages

ارفع الملفات التالية إلى المستودع ثم فعّل GitHub Pages من إعدادات المستودع:

- `index.html`
- `assets/css/style.css`
- `assets/js/*.js`
- `README.md`

لا يحتاج التطبيق إلى Backend أو إعدادات بناء.

## حدود النسخة الحالية

هذه نسخة Front-End محلية/Prototype. الصلاحيات وتسجيل الدخول تعمل داخل المتصفح فقط ولا تعادل حماية Backend رسمية. عند الاستخدام الرسمي متعدد المستخدمين يوصى بتحويلها إلى منظومة Backend مع مصادقة مركزية وقاعدة بيانات خادمية وسجل عمليات غير قابل للتلاعب.
